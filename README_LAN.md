# LAN対戦版 大富豪

同じWi-Fiまたは同じLANに接続したPC同士で対戦する版です。
インターネット上の不特定多数とは接続せず、ホストPCに参加者のPCが直接接続します。

## できること

- 最大6人まで対戦できます。
- ホストがロビーで対戦人数を2〜6人から選べます。
- 選んだ人数に足りない席はCPUが自動で担当します。
- 例：対戦人数を6人にして3台だけ接続すると、人間3人 + CPU3人で開始します。
- 例：対戦人数を4人にして1台だけ接続すると、人間1人 + CPU3人で開始します。

## 準備

各PCに同じプロジェクト一式を置き、Pygameを入れてください。

```powershell
uv venv --python 3.12 .venv
.\.venv\Scripts\Activate.ps1
uv pip install -r requirements.txt
```

すでに `.venv` があり、通常版が起動できるPCではこの作業は不要です。

## ルール設定

LAN対戦では、ホストPCの `settings.json` に保存されているルールとCPU難易度を使います。

1. ホストPCで通常版を起動します。

```powershell
python main.py
```

2. ゲーム設定でルールやCPU難易度を選びます。
3. 「保存してタイトルへ」を押して終了します。
4. その後、LANサーバーを起動します。

参加側PCの `settings.json` は対戦ルールには影響しません。

## 起動手順

一番簡単な方法は、`start_daifugo.bat` をダブルクリックする方法です。

- ホストPC：「ホストとして開始」を押します。
- 参加者PC：ホストPCのIPv4アドレスと名前を入れて「参加する」を押します。
- ホストPCは、ランチャー内の「ホスト用ローカルルール」でルールとCPU難易度を設定できます。

うまく開けない場合だけ、下のPowerShell手順で起動してください。

### ランチャーでの正しい順番

1. 全員が同じWi-Fiまたは同じLANに接続します。
2. ホストPCで `start_daifugo.bat` を開きます。
3. ランチャー上部に表示される接続用IPを確認します。
4. ホストPCで名前、人数、ローカルルールを設定します。
5. ホストPCで「ホストとして開始」を押します。
6. ホストのゲーム画面がロビーに入ったことを確認します。
7. 参加者PCで `start_daifugo.bat` を開きます。
8. 参加者PCの「ホストIP」に、ホストPCの接続用IPを入力します。
9. 参加者PCで名前を入力して「参加する」を押します。
10. 全員がロビーに入ったら、ホストのゲーム画面で人数を確認して開始します。

例：ホストPCの接続用IPが `192.168.11.53` の場合、参加者PCはホストIP欄に `192.168.11.53` と入力します。

### 1. ホストPCでIPアドレスを確認

PowerShellで次を実行します。

```powershell
ipconfig
```

使用中のWi-FiまたはEthernet欄にあるIPv4アドレスを確認します。
例：`192.168.1.25`

### 2. ホストPCでサーバーを起動

```powershell
python lan_server.py
```

または `start_lan_server.bat` をダブルクリックします。

初回はWindows Defenderファイアウォールの確認が出ることがあります。
信頼できるネットワークで、「プライベートネットワーク」を許可してください。

### 3. ホストPCもクライアントで参加

サーバーとは別のPowerShellを開きます。

```powershell
python lan_client.py --host 127.0.0.1 --name "ホスト"
```

または `start_lan_client.bat` を起動し、接続先に `127.0.0.1` を入力します。

### 4. 参加側PCでクライアントを起動

```powershell
python lan_client.py
```

画面で次を入力します。

- ホストPCのIPv4アドレス：例 `192.168.1.25`
- 自分の名前

## ロビーでの開始方法

- 最初に参加した人がホストになります。
- ホストだけが対戦人数を `-` / `+` で変更できます。
- 席に参加者がいない場合、その席はCPUになります。
- 「この人数で開始」を押すとゲームが始まります。

接続済みの人がいる席を対戦人数の外に出すことはできません。
たとえば席5に人がいる場合、対戦人数は最低5人になります。

## 操作

- カードをクリック：選択／解除
- 「出す」：選択したカードを出す
- 「パス」：パス
- `Enter`：出す／7渡し・10捨てを確定
- `P`：パス
- `Esc`：選択解除

7渡し・10捨てが発生した場合は、対象プレイヤーの画面だけが選択画面になります。
CPU席の場合は自動で処理されます。

## 6人の配札

ジョーカーありでは山札が53枚なので、6人に均等配布できません。
LAN版の6人対戦では各プレイヤーへ8枚ずつ、合計48枚を配ります。
残り5枚は伏せ札として使用しません。
ただし開始に必要な♦3が伏せ札に入った場合は、配札へ戻します。

## 接続できない場合

- ホストPCで `lan_server.py` が起動しているか確認してください。
- 入力したIPv4アドレスが正しいか確認してください。
- すべてのPCが同じWi-Fiまたは同じLANにいるか確認してください。
- ポート `50000` がファイアウォールで遮断されていないか確認してください。
- 学校や公共Wi-Fiでは、端末同士の通信が禁止されている場合があります。

### `timed out` と表示される場合

参加者PCからホストPCに到達できていません。次の順番で確認してください。

1. ホストPCで「ホストとして開始」を押した後、ホスト自身のゲーム画面がロビーに入っているか確認します。
2. 参加者PCのホストIPが、ホストPCのランチャー上部に表示された接続用IPと同じか確認します。
3. 全員が同じWi-Fiまたは同じLANにいるか確認します。別のWi-Fi、スマホテザリング、VPNでは接続できないことがあります。
4. Windows DefenderファイアウォールでPythonの通信を許可します。許可画面が出たら「プライベートネットワーク」を許可してください。
5. 学校や公共Wi-Fiでは、端末同士の通信が禁止されている場合があります。その場合は別のルーター、有線LAN、または端末同士の通信が許可されたネットワークを使ってください。
